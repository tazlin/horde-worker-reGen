AI Horde Worker

This worker shares your GPU with the AI Horde to generate images for people around the world. It is free and open source, licensed under the GNU AGPL-3.0 (see the LICENSE file in this folder).

How to run it

Double-click horde-worker.cmd in this folder.

If you let the installer create an AI Horde Worker Start Menu (or desktop) shortcut, you can click that instead and never open this folder at all.

The first launch sets itself up: it downloads the software needed to run, Python and PyTorch, which takes a few minutes, then opens a dashboard in your web browser. A short wizard there walks you through the only things you need to provide:

  • Your free AI Horde API key (get one at aihorde.net/register) and a worker name.
  • Which models to serve (a sensible default is chosen for your GPU).
  • An optional benchmark that tunes the worker for your machine.

When you click Start, your chosen models download and the worker begins earning kudos. The window you launched from is the worker: closing it stops the worker. Closing only the browser tab is fine, the worker keeps running and you can reopen the dashboard.

The curious: horde-worker.cmd opens the browser dashboard by default (--web). Run it with --terminal for an in-console text UI, or --headless for no UI at all.

Your configuration

Your settings (API key, worker name, and more) live in bridgeData.yaml in this folder. You do not need to edit it by hand: the dashboard's Config tab edits it for you. It is preserved across updates and reinstalls, so you only set it up once.

What are all these other files?

This folder holds every script for every way of running and maintaining the worker. You only ever need the one above. Here is what the rest are, so nothing here is a mystery. On Windows the .cmd files are the ones that apply; the matching .sh files are the Linux/macOS equivalents.

Run the worker

horde-worker
The main launcher. Opens the browser dashboard. This is the one to use.
horde-bridge
Runs the worker directly with no dashboard, using your bridgeData.yaml. For advanced/headless setups.
preload-models
Downloads your selected models ahead of time without starting the worker.

Keep it up to date

update
Full in-place upgrade to the latest release.
update-runtime
Re-syncs just the dependencies (a lighter update).

For AMD / ROCm on Linux only

*-rocm.sh
ROCm variants of the scripts above (for example horde-bridge-rocm.sh). Ignore these unless you run an AMD GPU on Linux.

If something goes wrong

create-logs-bundle
Packages your recent logs into a single file to attach when asking for help.

Behind the scenes

You should not need to run any of these directly; the launchers above use them for you.

runtime, bootstrap.py
Set up and manage the private Python environment.
run_worker.py, download_models.py, convert_config_to_env.py
Internal helpers the launchers call.
detect-backend.ps1
Detects your GPU so the right PyTorch build is installed.
bin\, .venv\
The package manager and the installed environment. Leave them alone.
